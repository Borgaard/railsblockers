Things Overemphasized in Picking Up RoR

Many resources (Hartl, Agile Web Development with Rails) put emphasis on the total devops workflow.


* Git/Version Control
* Test-Driven Development (RSpec, Spork, Cucumber)
* Javascript
* JQuery
* AJAX
* Coffeescript
* HAML or any other templating
* Environment setup, RVM or RBENV
* Your IDE or Editor, textmate or sublime text
* CSS, including Twitter Bootstrap
* SASS
* JSON, YAML
* Factories
* Agile workflow
* Bundler
* Rake

This is good form and you SHOULD learn it...

Good form...

...but bad for learning the ESSENTIALS for Rails and the web stack.

What SHOULD be emphasized (IMHO)

Getting data in and out of your web application

  • Routes
  • How to GET and POST data from the web to your app
  • How to manipulate data in a database from your app
  • How 3 items above form Model-View-Controller

  • It turns out Sinatra allows you to do this in a single page (well, maybe two pages)
